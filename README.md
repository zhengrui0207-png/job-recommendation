# Job Recommendation

一个基于 Boss 直聘抓取结果的岗位推荐页，用来把原始职位数据整理成更适合社招用户快速浏览、筛选和投递的列表。

当前页面地址：
[https://zhengrui0207-png.github.io/job-recommendation/](https://zhengrui0207-png.github.io/job-recommendation/)

## 项目结构

- `index.html`
  前端展示页，负责岗位过滤、匹配评分、卡片渲染、简历优化弹窗。
- `jobs.js`
  前端直接加载的职位数据文件，由抓取脚本自动生成。
- `scripts/daily-job-scrape.sh`
  实际是 Python 脚本，负责调用 `opencli` 搜索 Boss 职位、抓取详情、生成 `jobs.js` 和 `scripts/job_data.json`。
- `scripts/job_data.json`
  原始职位 JSON 备份。

## 如何执行抓取

先确保以下条件满足：

- 已安装 `opencli`
- `opencli doctor` 通过
- 浏览器扩展 `opencli Browser Bridge` 已连接
- Boss 直聘已登录
- 当前激活页是普通网页，不要停留在 `chrome://extensions` 或其他扩展页面

执行命令：

```bash
cd /Users/zhengrui/.newmax/workspace/job-recommendation
python3 scripts/daily-job-scrape.sh
```

脚本会自动完成这些动作：

1. 搜索岗位列表
2. 逐条补抓职位详情页
3. 生成 `jobs.js`
4. 更新 `scripts/job_data.json`
5. 自动提交并推送到 `main`

如果本次抓取结果为 `0`，脚本不会覆盖已有 `jobs.js`，避免线上页面被清空。

## 当前筛选规则

页面展示岗位时，会先做一轮硬过滤，目标是只保留更适合 1-3 年社招候选人的职位。

直接排除：

- 职位名包含 `实习`
- 职位名包含 `应届`
- 职位名包含 `外派`
- 职位名包含 `校招` / `管培`
- 学历要求包含 `博士`
- 经验要求明显大于 3 年，例如 `3-5年`、`5-10年`、`10年以上`
- HR 明显超过 3 天未活跃
- 公司规模明确低于 `100` 人

保留但不额外加分：

- `经验不限`
- `1年以内`
- HR 活跃度只显示为 `本周活跃` 这类不够精确的岗位
- 公司规模或融资信息暂时缺失的岗位

## 当前匹配算法

岗位排序按加分模型完成，页面展示分数时会把原始加分结果映射到更直观的 `40-98` 区间，避免所有岗位都挤在 90 分以上。

### HR 活跃度

- `今日活跃` / `刚刚活跃`：`+15`
- `3天内活跃`：`+15`

### 工作地点

- `北京·朝阳`：`+15`
- `北京（非朝阳）`：`+10`
- `杭州`：`+8`
- `上海`：`+5`
- `深圳 / 广州`：`+3`
- 其他城市：`-10`

### 公司规模

- `1000 人以上`：`+10`

### 融资阶段

- `B轮及以上` / `已上市`：`+10`
- `A轮` / `天使轮`：`-10`

### 公司类型

- AI Native 公司，例如 `MiniMax`、`智谱`、`Kimi`、`月之暗面`、`DeepSeek`、`百川`：`+20`

### 职位标题

- 标题包含 `AI` / `人工智能` / `大模型` / `LLM`：`+20`
- 标题包含 `产品经理`：`+15`
- 标题包含 `策略` / `增长`：`+10`
- 标题包含 `运营`：`+8`

### 技能匹配

- 每命中 1 个简历关键词：`+5`
- 该项最高：`+30`

### 薪资

- 薪资下限 `>= 15K`：`+5`

### 经验

- 经验明确为 `1-3年`：`+5`

## 数据说明

抓取结果会尽量补齐这些字段：

- `description`
- `welfare`
- `activeTime`
- `size`
- `finance`
- `skillKeywords`
- `jdSummary`
- `highlightPoints`

页面会优先展示：

- 技能命中
- 公司规模
- 融资阶段
- JD 摘要
- 关键要求高亮

## 已知限制

- 这个项目依赖 Boss 页面登录态和浏览器扩展桥接，`opencli` 连接异常时抓取会失败。
- Boss 搜索结果本身可能混入 `经验不限`、`本周活跃` 这类信息不够精确的岗位，所以当前页面会进一步做二次过滤和评分。
- 如果想稳定拿到更多完全符合规则的岗位，需要继续优化抓取关键词和抓取批次，而不只是改前端排序。
